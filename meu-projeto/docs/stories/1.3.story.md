# Story 1.3: App Motorista com Evidencia de Entrega

**Epic:** EPIC-MVP
**Status:** Ready
**Agent:** @dev
**Estimativa:** 2 dias

---

## Objetivo

Completar o app do motorista para capturar GPS na entrega. A captura de foto e assinatura JA FUNCIONA. Falta GPS e melhorar a UX de conclusao.

## Acceptance Criteria

### AC1: Captura de GPS
- [ ] Usar navigator.geolocation.getCurrentPosition() ao marcar parada como visitada
- [ ] Salvar latitude, longitude, accuracy no manifest_stops
- [ ] Nova migration adicionando colunas lat/lng/accuracy ao manifest_stops
- [ ] Fallback gracioso se usuario negar permissao (salvar sem GPS, mostrar aviso)

### AC2: Resumo de Entrega
- [ ] Apos completar todas as paradas, mostrar resumo:
  - Total de paradas visitadas vs puladas
  - Fotos coletadas
  - Assinaturas coletadas
  - Tempo total da rota
- [ ] Botao "Finalizar Rota" (ja existe completeManifest)

### AC3: Indicadores Visuais por Parada
- [ ] Badge de evidencia em cada parada: foto, assinatura, GPS
- [ ] Cor: verde (completa), amarelo (parcial), cinza (pendente)
- [ ] Timestamp visivel de quando foi marcada

### AC4: Historico de Rotas
- [ ] Pagina /driver/history com manifestos anteriores
- [ ] Filtro por data
- [ ] Resumo de cada rota (paradas, tempo, km estimado)

## Technical Notes

- GPS: API nativa do browser, sem custo
- Migration: ALTER TABLE manifest_stops ADD COLUMN latitude FLOAT, longitude FLOAT, gps_accuracy FLOAT
- Historico: nova page + action getDriverManifestHistory()
- Photo capture ja funciona (Supabase Storage)
- Signature pad ja funciona
