# Story 1.1: Fortalecer Setores de Producao

**Epic:** EPIC-MVP
**Status:** InProgress
**Agent:** @dev
**Estimativa:** 3 dias

---

## Objetivo

Tornar os setores de Secagem, Passadoria e Expedicao usaveis para operacao real em tablet. Hoje usam GenericSectorForm que e funcional mas generico demais. Precisamos de UX especifica por setor mantendo a arquitetura atual.

## Acceptance Criteria

### AC1: Secagem - Campos Especificos
- [x] Campo de selecao de equipamento (secadora) - JA EXISTE
- [x] Temperatura: low/medium/high com icones visuais - JA EXISTE (melhorado com temps)
- [ ] Novo: Estimativa de tempo baseada no peso (se disponivel do setor anterior)
- [x] Novo: Checkbox de "Filtro limpo" (pre-check obrigatorio)
- [x] Novo: Indicador de ciclo atual baseado no shift_cycles - JA EXISTE
- [x] KPIs no topo: pedidos secados hoje, tempo medio, SLA %

### AC2: Passadoria - Rastreio por Peca
- [x] Lista de pecas do pedido com checkbox individual (marcar como passada)
- [x] Contador visual: "12/15 pecas passadas" com progress bar
- [ ] Campo de observacao por peca (opcional - para danos/problemas)
- [x] Alerta se nem todas as pecas foram marcadas ao tentar concluir
- [x] KPIs: pecas passadas hoje, tempo medio, SLA %

### AC3: Expedicao - Notificacao ao Cliente
- [x] Selecao de embalagem (bag/box/hanger/other) - JA EXISTE
- [x] Selecao de manifesto - JA EXISTE
- [ ] Novo: Conferencia final de itens (checklist visual das pecas)
- [x] Novo: Botao "Notificar Cliente" pos-conclusao - JA EXISTE (WhatsApp dialog)
- [ ] Novo: Alerta SLA visual se pedido excedeu promised_at
- [x] Auto-gerar conta a receber - JA EXISTE

### AC4: KPIs em Todos os Setores
- [x] Barra de KPI no topo de cada setor (padrao: pedidos hoje, tempo medio, SLA %)
- [x] Cor do SLA: verde (no prazo), amarelo (proximo), vermelho (atrasado)
- [x] Dados via server action dedicada por setor

## Technical Notes

- Manter GenericSectorForm como base e estender via props condicionais
- Reusar complete_sector RPC (ja atomica)
- KPIs: criar action getSectorKpis(unitId, sector) generalizada
- Notificacao WhatsApp: gerar link `https://wa.me/{phone}?text={message}` (sem API)
- SLA calc: comparar NOW() com orders.promised_at
