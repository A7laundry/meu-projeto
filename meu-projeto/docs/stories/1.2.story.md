# Story 1.2: PDV com Fluxo de Pagamento Completo

**Epic:** EPIC-MVP
**Status:** Ready
**Agent:** @dev
**Estimativa:** 2 dias

---

## Objetivo

Completar o PDV da loja para que seja funcional para operacao diaria. O fluxo de criacao de pedido ja funciona. Faltam: recibo, historico de pedidos no PDV, e edicao/cancelamento.

## Acceptance Criteria

### AC1: Recibo Digital
- [ ] Apos criar pedido, exibir modal de recibo com: numero, itens, total, data prometida
- [ ] Botao "Imprimir" (window.print com CSS @media print)
- [ ] Botao "Enviar por WhatsApp" (link wa.me com resumo)
- [ ] Botao "Novo Pedido" para resetar

### AC2: Consulta de Pedidos no PDV
- [ ] Aba "Comandas" no PDV sidebar (ja existe /store/comandas)
- [ ] Busca por numero do pedido ou nome do cliente
- [ ] Status visual (badge colorido por status)
- [ ] Clique para ver detalhes + timeline de eventos

### AC3: Cancelamento de Pedido
- [ ] Botao "Cancelar Comanda" em pedidos com status 'received' apenas
- [ ] Motivo obrigatorio
- [ ] Adicionar status 'cancelled' ao order machine
- [ ] Registrar evento de cancelamento no audit log

### AC4: Abertura/Fechamento de Caixa
- [ ] Tela de abertura de caixa (valor inicial em dinheiro)
- [ ] Tela de fechamento (total vendido por metodo, diferenca)
- [ ] Tabela cash_register (nova migration)
- [ ] KPI no dashboard store: total do dia, meta vs realizado

## Technical Notes

- Recibo: componente React com CSS print-only, sem dependencia de impressora
- WhatsApp: link formatado, sem API
- Cancel: novo status no ORDER_TRANSITIONS + migration para CHECK constraint
- Cash register: migration simples (open_amount, close_amount, date, unit_id)
